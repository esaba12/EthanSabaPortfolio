  <action>
    Create src/app/lib/motion-utils.ts (new `lib` directory alongside the existing
    `components` directory under src/app/). Mark it `'use client'` (every consumer is a
    Client Component; GSAP plugin registration is grouped here per the UI-SPEC's
    prescriptive Integration Pattern: "register gsap.registerPlugin(ScrollTrigger) once at
    module scope in a shared motion-utils file, not per-component"). Import `gsap` from
    `'gsap'`, `{ ScrollTrigger }` from `'gsap/ScrollTrigger'`, `{ SplitText }` from
    `'gsap/SplitText'`, `{ DrawSVGPlugin }` from `'gsap/DrawSVGPlugin'`, and `{ useGSAP }`
    from `'@gsap/react'`. Call `gsap.registerPlugin(ScrollTrigger, SplitText, DrawSVGPlugin,
    useGSAP)` once at module scope (this is the single registration point for the whole
    site — no other file may call registerPlugin). Re-export `gsap`, `ScrollTrigger`,
    `SplitText`, `DrawSVGPlugin`, and `useGSAP` as named exports so consumers import
    everything from this one module instead of importing gsap/plugins directly. Also export
    a `prefersReducedMotion(): boolean` helper that returns `false` when `window` is
    undefined (SSR-safe) and otherwise returns
    `window.matchMedia('(prefers-reduced-motion: reduce)').matches` — this mirrors the
    pattern already used inline in WaveUnderline.tsx and centralizes it for reuse by every
    future GSAP effect, per the UI-SPEC's requirement that "every GSAP ScrollTrigger effect
    must check prefers-reduced-motion... and fall back to the final visual state."

    Create src/app/components/SmoothScrollProvider.tsx, marked `'use client'`. It wraps
    `ReactLenis` from `'lenis/react'` around its `children` prop with the `root` prop set
    (site-wide document scroll, not a scoped container) and `options={{ autoRaf: false }}`
    (disables Lenis's own internal rAF loop so GSAP's ticker drives it instead, per the
    UI-SPEC). Hold a ref to the Lenis instance via `useRef` passed as the `ref` prop on
    `<ReactLenis>`. In a `useEffect` with an empty dependency array: register a `gsap.ticker`
    callback that calls `lenisRef.current?.lenis?.raf(time * 1000)` on every tick (GSAP's
    ticker passes time in seconds; Lenis expects milliseconds, hence the `* 1000`), call
    `gsap.ticker.lagSmoothing(0)` to disable lag smoothing, and return a cleanup function
    that removes the ticker callback. Additionally call `useLenis` from `'lenis/react'` with
    a callback that invokes `ScrollTrigger.update()` on every scroll event, so ScrollTrigger
    stays in sync with Lenis's smoothed scroll position (equivalent to the UI-SPEC's
    `lenis.on('scroll', ScrollTrigger.update)` requirement, expressed via the React hook).
    Import `gsap` and `ScrollTrigger` from the motion-utils module created above (relative
    import `../lib/motion-utils`), not from `gsap` directly, so plugin registration always
    happens before this component runs. This component only creates the provider — it is
    not yet mounted anywhere; Plan 01-02 mounts it in layout.tsx.
  </action>
