# Phase 1 — UI Design Contract

> Visual and interaction contract for the "Engineering Blueprint" design system and Home page rebuild. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Aesthetic direction (explicit commitment):** Retro-futuristic technical / brutalist-precision hybrid — "Engineering Blueprint." Charcoal canvas, drafting-grid, annotation typography, one committed signal-amber accent, cinematic GSAP-driven reveals. This is NOT soft-SaaS, NOT glassmorphism, NOT purple-gradient, NOT rounded-card-everywhere.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom hand-rolled Tailwind design system (no shadcn/Radix) |
| Preset | not applicable |
| Component library | none (bespoke primitives defined below in Component Inventory) |
| Icon library | `@heroicons/react` (nav menu icons) + `lucide-react` (footer social icons) — existing, keep both, do not add a third |
| Font | Display: **Chakra Petch** (Google Font, weights 300–700, technical/schematic cut-corner letterforms) · Body/Heading: **Geist Sans** (already installed) · Mono/Label: **Geist Mono** (already installed) |

**Rationale for "Tool: none":** `components.json` is absent and this is a Next.js/React project, which normally triggers the shadcn gate. shadcn was deliberately declined for this phase: the locked aesthetic direction depends on bespoke structural motifs (corner brackets, coordinate labels, hairline rules, a persistent blueprint grid) that a generic component registry would fight against, and PROJECT.md's Key Decisions table only names framer-motion + GSAP/ScrollTrigger + Lenis as the stack — shadcn/Radix are never mentioned. Introducing a component registry here would push the UI toward the generic "AI slop" look this milestone explicitly exists to remove. Registry Safety Gate is therefore not applicable for this phase.

**Font swap note:** `layout.tsx` currently loads `Space_Grotesk` into the `--font-display` CSS variable but the current Home page never uses it (the h1 uses a gradient-text utility instead). This phase replaces that unused Space Grotesk import with `Chakra_Petch` in the same `--font-display` variable slot — no new CSS variable name needed, just a font swap at the import site.

## Spacing Scale

Declared values (must be multiples of 4), confirmed against the existing `container` padding scale in `globals.css`:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, coordinate-label internal padding |
| sm | 8px | Compact element spacing, corner-bracket inset from panel edge |
| md | 16px | Default element spacing, badge/tag padding |
| lg | 24px | Card/panel internal padding, section label spacing |
| xl | 32px | Layout gaps between stacked blocks, blueprint grid cell size |
| 2xl | 48px | Major section breaks (Intro → Skills → Projects) |
| 3xl | 64px | Page-level top/bottom spacing (hero top padding, section end padding) |

Exceptions:
- Icon-only touch targets (mobile nav hamburger, footer social icons) = 44px minimum hit area, not on the 8pt scale (accessibility floor, existing `w-10 h-10`/`p-2` combinations already satisfy this — keep as-is).
- Corner-bracket arm length = 24px (`lg` token), bracket stroke inset from the panel edge = 8px (`sm` token) — reused identically on every bracketed panel across the whole site, not just Home.
- Blueprint grid line pitch = 32px (`xl` token) on desktop, 24px (`lg` token) on mobile — see Grid & Canvas Pattern below.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (Geist Sans Regular) | 1.5 |
| Label | 13px | 400 (Geist Mono Regular) | 1.4 |
| Heading | 28px (desktop) / 22px (mobile, <640px) | 700 (Chakra Petch Bold) | 1.2 |
| Display | 64px (desktop, lg+) / 40px (mobile, <640px) | 700 (Chakra Petch Bold) | 1.05 |

Only two weight values are used sitewide: **400** (Geist Sans body copy, Geist Mono labels) and **700** (Chakra Petch display + heading). Do not introduce a third weight (no 500/600 "medium/semibold" anywhere in this phase).

Notes:
- Display line-height is intentionally tighter (1.05) than the template's general heading recommendation (1.2) — standard practice for very large (64px+) display type to avoid excess vertical whitespace between wrapped lines. Heading role (28px section titles like "Skills", "Featured Projects") keeps the more legible 1.2.
- Label role (13px mono) is used for: nav items are NOT mono (nav stays Geist Sans per existing pattern), but section index tags (`[ 01 ]`), coordinate/annotation readouts, Badge chips, and ProjectsGrid tag chips ARE mono. Uppercase + `tracking-wide` (0.04em letter-spacing) on Label role for the technical-readout feel.
- Existing `globals.css` currently sets body line-height to 1.6 sitewide — update to 1.5 to match this contract for consistency with the new type scale.
- Responsive scaling for Display and Heading should be implemented as two explicit Tailwind breakpoint values (not fluid `clamp()`), matching how the rest of the site already scales type (see existing `h1`/`h2` rules in `globals.css`), for implementation predictability.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (existing `brand.bg`) | Page background, canvas the blueprint grid sits on |
| Secondary (30%) | `#1a1a1a` (existing `brand.surface`), hover/lighter state `#2a2a2a` (existing `brand.surface-light`) | Panel/card fills (Skills panel, project cards), nav pill background, footer background |
| Accent (10%) | `#FF7A1A` ("signal amber" — new token, replaces existing `brand.accent` `#3b82f6` blue) | See explicit reserved-for list below only |
| Destructive | `#E5484D` (new token, `brand.danger`) | Reserved for future phases (contact form errors) — not used anywhere in Phase 1 Home page scope |

Accent (`#FF7A1A`) reserved for — this exact list, nothing else:
- Active/current nav link pill background and text (navbar + footer active state)
- WaveUnderline stroke beneath the hero headline (already amber-tinted via `brand-accent`; recolor to the new hex)
- Corner-bracket accent flourish + coordinate-label text color (the `[01]`, `X:/Y:` annotation motif) sitewide
- Hover/focus ring on interactive cards and buttons (ProjectsGrid cards, Badge focus ring, nav focus ring)
- The single-instance line-draw stroke used in the one big early scroll-triggered "wow" reveal on Home page load
- "Click to view project" indicator icon + text on ProjectsGrid cards (existing element, recolor from blue to amber)

Do not use accent for: body text, section headings, panel borders in their resting (non-hover) state, badge fill/background, or any decorative fill larger than a 2px stroke. Panel borders in resting state stay neutral (`brand-border` gray, existing `#374151`), consistent with a blueprint's fine hairline rules rather than a colored outline.

**Color migration note:** `brand.accent` (`#3b82f6` blue) and `brand.pulse` (`#06b6d4` cyan) in the existing `tailwind.config.js`, plus the hero's `from-cyan-300 via-blue-500 to-violet-600` gradient text utility, are the exact "generic gradient headline" the PROJECT.md context calls out for replacement. This phase removes the gradient utility from the h1 and repoints `brand.accent` to `#FF7A1A`. `brand.pulse` (cyan) is retired — no cyan/blue/violet hue should remain anywhere in the shipped Home page.

## Grid & Canvas Pattern

Not a template-default section, but required for this phase since the persistent blueprint grid is a core locked decision.

- Implementation: a fixed-position full-viewport `<div>` behind all page content (`position: fixed; inset: 0; z-index: 0; pointer-events: none`) rendering a repeating CSS `background-image` linear-gradient grid — NOT an SVG pattern re-rendered per section, one instance mounted once in `layout.tsx` so it is truly persistent and identical across all pages (including future Phases 2–5).
- Line color: `rgba(255,255,255,0.04)` on the `#0a0a0a` canvas — faint enough to read as texture, never competes with foreground content. Do not exceed 0.06 opacity.
- Pitch: 32px squares desktop (`xl` spacing token), 24px squares mobile (`lg` spacing token) — reuses the spacing scale rather than an arbitrary grid value.
- Optional accent: every 4th line (128px/96px major grid) may render at slightly higher opacity (`rgba(255,255,255,0.07)`) to mimic blueprint major/minor rule convention — implementer's discretion, not required for approval.
- The grid must NOT use the signal-amber accent color. It is a neutral texture only.

## Motion System

Required detail beyond the template default, since motion is this phase's primary success criterion.

**Stack (add to `package.json`):**
- `gsap` (includes `ScrollTrigger` plugin) — scroll-scrubbed choreography, line-draw reveals, pinned sections
- `@gsap/react` — `useGSAP()` hook for automatic effect cleanup in React components
- `lenis` — import the React wrapper from `lenis/react` (`<ReactLenis>` / `useLenis`), not the deprecated `@studio-freight/*` packages
- `framer-motion` stays exactly as-is for micro-interactions (hover/tap/focus transitions on Badge, nav pills, card hover) — do not migrate existing framer-motion usage to GSAP

**Integration pattern (prescriptive, for the executor):**
- Mount `<ReactLenis root>` once in `layout.tsx` (client component) wrapping `{children}`, so smooth scroll is sitewide and shared, not Home-page-specific.
- Register `gsap.registerPlugin(ScrollTrigger)` once at module scope in a shared motion-utils file, not per-component.
- Sync Lenis and ScrollTrigger: `lenis.on('scroll', ScrollTrigger.update)` plus drive Lenis off `gsap.ticker` (`gsap.ticker.add((time) => lenis.raf(time * 1000))`) rather than Lenis's own internal rAF loop, so both stay frame-synced.
- Every GSAP ScrollTrigger effect must check `window.matchMedia('(prefers-reduced-motion: reduce)')` before running (matching the existing pattern already used in `WaveUnderline.tsx`) and fall back to the final visual state with no animation.

**Choreography contract for Home page (one big moment, restraint elsewhere):**
1. **The one scroll-stopping moment (required):** on initial page load, before any scroll input, the hero headline reveals via a GSAP line-draw/mask animation (text masked by a horizontal wipe or clip-path reveal, not a fade) synced with the WaveUnderline stroke drawing in beneath it. Duration ~1.1–1.4s, easing `power3.out`. This is the single "wow" moment — do not repeat this treatment elsewhere on the page.
2. **Section entries (restrained):** Skills panel and Featured Projects section use simple ScrollTrigger-driven fade+8px-rise reveals (opacity 0→1, y 8px→0, `power2.out`, ~0.4s, triggered at `top 80%`) — same treatment for both sections, no per-section bespoke animation.
3. **Micro-interactions (framer-motion, unchanged mechanism):** Badge focus/hover, nav pill active-state transition, ProjectsGrid card hover-scale on image — keep framer-motion, keep existing durations (~0.2–0.3s) already in the codebase.
4. Do not animate: the blueprint grid background, footer, or nav on every scroll — these stay static/instant except their own existing hover states.
